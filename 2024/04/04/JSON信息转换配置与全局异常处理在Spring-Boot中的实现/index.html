JSON信息转换配置与全局异常处理在Spring Boot中的实现

2024-04-04 项目实战-仿B站 FastJson, HTTP消息转换器, 全局异常处理 0 评论 字数统计: 2.5k(字) 阅读时长: 10(分)

在构建Web应用时，合理地配置JSON信息转换和全局异常处理机制能够极大地提升系统的健壮性和用户体验。本文将详细介绍一个基于Spring Boot的项目中，如何通过JsonHttpMessageConverterConfig、ConditionException、CommonGlobalExceptionHandler以及JsonResponse类，实现JSON信息转换配置和全局异常处理。

一、JSON信息转换配置

1.1 JsonHttpMessageConverterConfig 类

JsonHttpMessageConverterConfig 类是一个Spring配置类，通过@Configuration注解表明其负责提供应用所需的Bean配置。

1.2 fastJsonHttpMessageConverter() 方法

该方法使用@Bean注解声明为一个Bean，返回类型为HttpMessageConverters，表示它将被用于Spring MVC的HTTP消息转换器链中。@Primary注解确保当存在多个同类型Bean时，该Bean优先被选择。

1.3 配置FastJsonHttpMessageConverter

在方法内部，我们创建了一个FastJsonHttpMessageConverter实例，它是Spring MVC与FastJson库结合使用的HTTP消息转换器，负责将Java对象序列化为JSON响应以及将JSON请求反序列化为Java对象。

首先，我们设置了FastJson的日期格式为"yyyy-MM-dd HH:mm:ss"，确保时间数据以统一、易读的格式返回给客户端。接着，我们配置了一系列SerializerFeature，如：

  • PrettyFormat: 使输出的JSON格式化，易于阅读。
  • WriteNullStringAsEmpty, WriteNullListAsEmpty, WriteMapNullValue: 当值为null时，分别将其序列化为空字符串、空列表和空映射，避免返回null值引发前端处理问题。
  • MapSortField: 对Map类型的键进行排序，保证JSON输出顺序的一致性。
  • DisableCircularReferenceDetect: 禁用循环引用检测，防止因对象间循环引用导致的序列化失败。

最后，我们将配置好的FastJsonConfig设置到FastJsonHttpMessageConverter实例中。

综上所述，JsonHttpMessageConverterConfig类通过配置FastJsonHttpMessageConverter，实现了对Spring Boot应用中JSON信息转换的定制化处理，确保了JSON响应的格式规范、易读以及与前端交互的友好性。

二、全局异常处理配置

2.1 ConditionException 类

ConditionException 是自定义的运行时异常类，继承自RuntimeException。它用于封装特定业务条件不满足时抛出的异常，包含一个code属性用于表示错误代码，便于在全局异常处理器中根据不同的错误代码生成对应的JSON响应。

2.2 CommonGlobalExceptionHandler 类

CommonGlobalExceptionHandler 类通过@ControllerAdvice注解标记为一个全局异常处理器，能够拦截所有控制器（@Controller）抛出的异常。@Order(Ordered.HIGHEST_PRECEDENCE)注解确保该处理器在其他同类处理器之前执行。

2.3 commonExceptionHandler() 方法

commonExceptionHandler() 方法使用@ExceptionHandler注解指定其处理所有Exception类型的异常。@ResponseBody注解表明该方法的返回值将直接作为HTTP响应体返回，无需视图解析。

方法内首先获取异常的详细信息，然后判断异常是否为ConditionException实例。如果是，则从异常中提取错误代码；否则，设定默认错误代码为"500"。最后，根据错误代码和异常消息构造一个JsonResponse<String>对象并返回。

三、JsonResponse 类

JsonResponse 类是一个泛型类，用于封装API接口的返回结果。它包含三个属性：code（错误代码）、msg（错误消息或成功提示）和data（实际业务数据）。类中提供了丰富的构造方法和静态工厂方法，便于快速创建不同场景下的响应对象。

四、文件之间的联系与应用串联

  1. JSON信息转换配置：

    • JsonHttpMessageConverterConfig 类配置了FastJsonHttpMessageConverter，使得Spring Boot应用在处理HTTP请求和响应时能自动将Java对象与JSON格式数据相互转换。
    • 这种配置确保了应用返回的JSON响应遵循预设的日期格式和序列化规则，提高了与前端交互的规范性和一致性。

  2. 全局异常处理配置：

    • 当控制器中发生异常时，CommonGlobalExceptionHandler 类的commonExceptionHandler() 方法会被调用，捕获并处理异常。
    • 根据捕获到的异常类型（尤其是自定义的ConditionException），生成包含错误代码和消息的JsonResponse对象，以统一的JSON格式返回给客户端。

  3. JsonResponse 类：

    • 作为API响应的通用数据结构，JsonResponse 类将错误代码、消息和业务数据封装在一起，便于前端根据错误代码进行相应的错误处理或展示成功结果。
    • 在全局异常处理器中，JsonResponse 被用来包装异常信息，形成标准化的JSON响应返回给客户端。

综上，这三个类共同构成了一个完整的JSON信息转换与全局异常处理体系，实现了Spring Boot应用中JSON数据的规范化转换以及异常情况的统一处理，提升了应用的健壮性和前后端交互体验。
